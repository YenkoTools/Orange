# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| latest  | :white_check_mark: |

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

If you discover a security vulnerability in this project, please report it responsibly:

1. Navigate to the **Security** tab of this repository.
2. Click **Report a vulnerability** to open a private advisory.
3. Provide a clear description of the vulnerability, including:
   - The type of vulnerability (e.g., injection, authentication bypass, data exposure)
   - Steps to reproduce the issue
   - Potential impact
   - Any suggested mitigations

### What to Expect

- **Acknowledgment**: You will receive an acknowledgment within **48 hours**.
- **Status updates**: You will be kept informed of the progress toward a fix.
- **Disclosure**: Once the vulnerability is fixed, we will coordinate a responsible disclosure timeline with you.

We appreciate your help in keeping this project and its users safe.
